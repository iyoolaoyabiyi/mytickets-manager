## Copy Deck and Wireframes

This document contains the full text copy and structured text‑based wireframes for all screens. It also includes global copy (header, footer, toasts, validations) and string IDs to help keep React/Vue/Twig in sync.

## Global

### App Name

- **myTickets Manager**

### Route Map

- `/` Landing
- `/auth/login` Login
- `/auth/signup` Signup
- `/dashboard` Dashboard
- `/tickets` Tickets List, Ticket Create/Edit

### Auth Guard

- Protected: `/dashboard`, `/tickets`
- If no `ticketapp_session`: redirect to `/auth/login` and show session‑expired message.

### Status Colors

- `open` -> Green tone
- `in_progress` -> Amber tone
- `closed` -> Gray tone

### Global Components (copy)

- **Header:**
  - Brand: `myTickets Manager`
  - Auth links (unauth): `Login` · `Get Started`
  - Auth links (auth): `Dashboard` · `Tickets` · `Logout`
  - Accessible nav toggle label (mobile): `Toggle navigation`
- **Footer:**
  - Left: `© ${currentYear} myTickets Manager`
  - Right links: `Privacy` · `Terms` · `Help`
  - Small note: `This is a demo application for learning and testing.`
- **Toasts & Inline Alerts:**
  - Success: `Done. Your changes have been saved.`
  - Create success: `Ticket created successfully.`
  - Update success: `Ticket updated successfully.`
  - Delete success: `Ticket deleted.`
  - Auth success: `Welcome back {name}.`
  - Session expired: `Your session has expired, please log in again.`
  - Invalid session: `Invalid session, please login.`
  - Load error: `Failed to load tickets. Please retry.`
  - Network error: `A network error occurred. Please check your connection and try again.`
  - Auth error: `Invalid email or password.`
  - Validation error (generic): `Please fix the errors and try again.`

### Validation Copy

- Field required: `This field is required.`
- Email invalid: `Enter a valid email address.`
- Password length: `Use at least 8 characters.`
- Passwords mismatch: `Passwords do not match.`
- Title required: `Enter a ticket title.`
- Status required: `Select a status.`
- Description length: `Description is too long.`

### Empty States

- Tickets list empty (first‑time): `No tickets yet.` with action `Create your first ticket`.
- Tickets list filtered empty: `No tickets match your filters.` with action `Clear filters`.

### Common Microcopy

- Search placeholder: `Search tickets…`
- Filter label: `Status`, `Priority`
- Primary button: `Save`
- Secondary button: `Cancel`
- Destructive button: `Delete`
- Confirm modal title: `Delete ticket?`
- Confirm modal body: `This action cannot be undone.`
- Confirm modal actions: `Delete` · `Cancel`

## Notes for Layout Consistency

- Max width 1440 px, centered on large screens.
- Hero with wavy bottom edge and at least one overlapping circle.
- Use card‑style boxes for stats and tickets.
- Mobile: stacked layout with collapsible navigation. Tablet/Desktop: multi‑column grid with consistent spacing.
- Semantic HTML, alt text, visible focus states, sufficient color contrast.
