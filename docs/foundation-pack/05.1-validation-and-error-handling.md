## Global rules (source of truth)

- Toast text and validation messages come from your copy deck (`toast.*`, `validation.*`).
- Guarded routes: `/dashboard`, `/tickets` -> if no `ticketapp_session`, redirect to `/auth/login` and show `toast.session` message.

## Auth: Login

**Fields & rules**
- Email: valid email; required -> `validation.email`, `validation.required`.
- Password; required -> `validation.required`.
**Inline errors**: under each field via `c-field__message`; disable submit until valid.

**Toasts**
- Success -> `toast.auth.success` then route -> Dashboard.
- Invalid credentials (401) -> `toast.auth.error`.
- Network/unknown -> `toast.network.error`.

**States**
- `idle` -> `submitting` -> `success|error` (error also maps field-level or generic toast).

## Auth: Signup

**Fields & rules**
- Name: required; `validation.required`.
- Email: required, valid email; `validation.email`.
- Password: required min 8; `validation.password.length`.
- Confirm password: required, matches password; -> uses `validation.password.match`.

**Toasts**
- Success -> `toast.postsignup` then redirect -> Login (no auto-login by design).
- Email exists (409) -> `toast.validation`. 
- Network -> `toast.network.error`.
  

## Route guard / session check (every page load & on actions)

- If **no token** -> redirect to `/auth/login` + `toast.session.expired` (or `toast.session.invalid` for malformed).
- If **expired token** -> clear token -> same flow as above.

## Tickets: Filters/Search (list view)

- Search: trim, allow empty; (optionally) max 100 chars; no toast (silent).
- Status: one of `all|open|in_progress|closed`.
- Priority: one of `high|medium|low`.

## Tickets: Create/Edit (modal on `/tickets`)

**Fields & rules**
- Title: required (1â€“100 chars) -> `validation.title.required`.
- Description: optional, max 1000 -> `validation.description.length`.
- Status: required enum `open|in_progress|closed` -> `validation.status.required`.
- Priority: optional enum `high|medium|low`  -> `validation.priority.required`.
**Inline errors**: under fields; disable Save until valid; `aria-describedby` for IDs.  

**Toasts**
- Create success -> `toast.create.success`.
- Update success -> `toast.update.success`.
- Server 400/422 -> `toast.validation`.
- 404 on update/delete (stale) -> `toast.load.error`.
- Network -> `toast.network.error`.

## Tickets: Delete (confirm modal)

- Confirm -> on success `toast.delete.success`.
- 404 (already deleted) -> `toast.ticket.error`.
- Network -> `toast.network.error`.

## Tickets: Load (list & dashboard stats)

- Success: render.
- Error: show `toast.load.error`; show empty/error state as appropriate.
